Disabled state

Do's
  • Use verbs for button labels (eg. "Save" instead of "OK")
  • Where possible limit each screen to a single primary action. This helps to guide the user to complete a task.
  • Use the disabled state when a prior action needs to be taken before the button becomes enabled
  • Icon on the button should clearly communicate what action will occur
Don'ts
  • Do not use the disabled state if a user does not have permission to perform the action
  • Avoid compound actions (eg. "Save and Close" )
Rules
  • At least 2 items
  • No more than 4 items
  • Even width for all items

Progress bar

Progress bars make long processes tolerable

Users are more willing to tolerate a long wait time if they see a progress bar.

Don’t use static progress indicators.

Spinner

Spinner is for short processes less than 4 seconds.

Spinners don’t tell users how long the process will take to load. If you use it for long processes, users will end up wondering if something went wrong with the app.

If a process takes longer than 4 seconds to load, you should use a Progress Bar.

Use .spinner-fixed-center-vh class on spinner will make the spinner absolute fixed position in the center of view port.

Tooltips are brief pieces of information that appear on hover state over an element to clarify the meaning or use of an interaction element for the user.

Tooltips help users understand unknown or unfamiliar objects that aren’t described directly in the user interface.

They are a powerful way to simplify UI — provide information users need when they need it (reduce screen clutter).
